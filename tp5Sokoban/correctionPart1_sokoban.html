# Eléments de réponse pour le TP Sokoban <img src="https://upload.wikimedia.org/wikipedia/commons/4/4b/Sokoban_ani.gif?20170729191043"> source [wikipedia](https://fr.wikipedia.org/wiki/Sokoban) - Les règles du jeu : <https://fr.wikipedia.org/wiki/Sokoban> ## partie 1 : en Prolog * On peut utiliser Prolog * 4 parties : 1. description de la carte 2. description du jeu sokoban 3. description d'un problème de planification 4. prédicats utilitaires * pour la conception, on va partir du plus général et aller vers le plus particulier (3 -> 2 -> 1) * le script Prolog ira lui du plus particulier vers le plus général (1 -> 2 -> 3) * les utilitaires finiront par se glisser entre 1 et 2. ### un problème de planification * un plan = liste d'actions (vrai, mais peu descriptif) * un plan = un chemin dans l'espace des états. Les actions représentent les arcs de ce chemin, les états sont les noeuds * on peut écrire cette description à l'aide d'un prédicat ternaire `planValide(Actions, Etats, Longueur)` ```prolog planValide([], [_], 0). planValide([A|AT], [S1, S2 | ST], N) :- N > 0, N1 is N-1, do(A, S1, S2), planValide(AT, [S2 | ST], N1). ``` ### les règles de Sokoban * il s'agit ici de décrire les actions spécifiques à Sokoban * bien que ce ne soit pas strictement nécessaire, il est agréable de décomposer cette description en une partie "physique" et une partie centrée sur le personnage #### partie physique on définit un prédicat ternaire `positionRelative(Direction, Case1, Case2)` qui définit la relation de proximité entre les cases. La variable `Direction` prend une valeur parmi les constantes `up, down, right, left`. Les variables `Case1` et `Case2` sont filtrées à l'aide de la fonction binaire `pos(X,Y)` positionRelative(up, pos(X,Y), pos(X1,Y1)) :- case(X,Y), case(X1,Y1), X = X1, Y1 is Y+1. ... Le prédicat `case(X,Y)` vérifie que la position (X,Y) correspond effectivement à une case du plateau de jeu (cf. partie 3) #### partie centrée sur le personnage * on décrit les actions qu'il va pouvoir entreprendre à l'aide d'un prédicat ternaire `do(Action, EtatOrigine, EtatDestination)`) * on peut aussi décomposer l'ensemble des actions comme un produit cartésien $Verbes \times Directions$. Ainsi, une action sera filtrée par la *fonction* binaire `act(Verbe,Direction)` * chaque état doit encoder les **fluants**. Ici, il s'agit de la position du personnage ainsi que celles des caisses. Un état sera filtré par la fonction `state(me(X,Y), boxes(L))` où `L = [pos(X1,Y1), ... ,pos(XN,YN)]` est la liste des positions des caisses ```prolog do(act(move,D), state(me(X,Y) , boxes(L)), state(me(X1,Y1), boxes(L))) :- positionRelative(D, pos(X,Y), pos(X1,Y1)), notIn(pos(X1,Y1), L). * Par rapport à la description en STRIPS, le prédicat `do` traite simultanément les préconditions et les effets * on utilise un prédicat utilitaire binaire `notIn(X,L)` qui s'efface lorsque l'élément $X$ ne figure pas dans la liste $L$ ```prolog notIn(_,[]). notIn(X, [H|T]) :- dif(X,H), notIn(X,T). ``` ```prolog do(act(push,D), state(me(X,Y) , boxes(L)), state(me(X1,Y1), boxes(pos(X2,Y2)|L))) :- positionRelative(D, pos(X,Y), pos(X1,Y1)), positionRelative(D, pos(X1,Y1), pos(X2,Y2)), trouveEnleve(pos(X1,Y1),L,L1) notIn(pos(X2,Y2), L). ``` Le prédicat utilitaire `trouveEnleve(X,L,L1)` s'efface ssi la liste $L$ contient $X$ et $L1$ est la liste obtenue en suppriant de $L$ la première occurence de $X$. ```prolog trouveEnleve(X, [X|T], T). trouveEnleve(X, [Y|T], [Y|T1]) :- dif(X,Y), trouveEnleve(X,T,T1). ``` #### conditions de victoire Un état final est victorieux ssi toutes les caisses occupent des positions cibles (repérées à l'aide d'un prédicat unaire `target`) ```prolog allOnTarget([]). allOnTarget([H|T]) :- target(X), allOnTarget(T). victory(state(_,boxes(L))) :- allOnTarget(L). ``` ### la carte chaque niveau est décrit par un plateau de jeu, contenant des cases accessibles, des cases cibles, ainsi que par un état initial. ```prolog case(2,5). case(3,5). case(X,4) :- between(2,5,X). target(pos(1,3)). initial(state(me(2,4), boxes([pos(2,5), pos(3,5)]))) ``` ### les requêtes * on peut vouloir trouver tous les plans de taille $N$ à partir d'un état donné ```prolog -? Etats = [X|_], initial(X), planValide(Etats, Actions, 3) ``` * on aimerait pouvoir trouver tous les plans de taille donnée se terminant en un état donné. Malheureusement, le fonctionnement de `is` ne le permet pas (on pourrait redéfinir l'arithmétique avec un prédicat `next`...) * on aimerait trouver un plan victorieux de taille donnée ```prolog -? Etats = [X|_], initial(X), planValide(Etats, Actions, 3), endsWith(Y,Etats), victory(Y) ``` où le prédicat utilitaire `endsWith(X,L)` s'efface lorsque la liste L se termine par X ```prolog endsWith(X,[X]). endsWith(X,[_|T]) :- endsWith(X,T). ``` Malheureusement, en pratique, l'algorithme de backtracking de Prolog ne permet pas de découvrir des plans victorieux de taille 15 ou plus
